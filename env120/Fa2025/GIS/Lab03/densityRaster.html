Creating a density raster

Let’s say we wanted to stay at one of the huts on Kapiti Island for a birdwatching trip, and wanted to know where we were likely to see the most birds. To figure this out, we can use a set of bird sightings to learn which hut features the most bird sightings.

  • In the Table of Contents, click the box next to the elevation and huts data to hide them.

  • Load the kapiti_birds dataset. These are actual locations of bird sightings on Kapiti Island obtained from iNaturalist (via gbif.org). It should look a bit like this:

  • Go back to the toolboxes. Expand the Spatial Analyst Tools toolbox and the Density toolset, and select the Kernel Density tool. Kernel density is a measure of the density of objects calculated for a moving window over the area of interest. This allows us to visualize how density varies over space.

  • Select the kapiti_birds as the Input point or polyline features. Leaving the Population field value as NONE will make each feature count once; if you had an attribute in your data that counted the number of objects at a given location (for example, if each point had an attribute for the number of birds observed at that location), you could select that field to incorporate those numbers. For the Output raster, save it as birdDensity.

  • Since we’re looking at data recorded on an island, we may not want to include ocean spaces in our calculation. To change this, select the Input barrier features menu in the and choose the kapiti_outline dataset.

  • We may also want to limit the output to the island rather than have our estimates extend beyond its boundaries. To do this, click on Environments at the top of the pane. Under Mask, select the kapiti_outline dataset. This will ‘mask’ any values that are generated outside of the island outline polygon. When you have this finished, click Run. The output should look like this:

  • If you look at the left, you can see that the density values are shown using a color gradient from transparent to dark purple. For each cell, this is the density of bird sightings per unit area (km2). You can see how these vary with respect to the hut locations by using the Explore tool.

  • Right click on the birdDensity dataset in the Table of Contents to look at the Symbology. You can see that rather than a “stretched” color ramp, these are “Classified”, which means that values are grouped and assigned a color. This is why there are visible contours between density values on the map.

  • Under Method, notice that it says Equal Interval. This means that the data are grouped in equally sized intervals from the minimum to the maximum value. You can choose from a few different options here. A commonly-used method is the Natural Breaks or Jenks method, which divides the data into groups that aim to maintain a consistent level of variability between them. You can also modify the number of different groups by changing the Classes value.

Try it yourself!

Using the skills above and what you’ve already learned, create a map that shows the elevation data with 50% transparency, and shows the huts as proportional or graduated symbols indicating the relative density of bird sightings at their locations.

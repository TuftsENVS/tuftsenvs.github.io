2  Transforming data

Beyond subsetting, other functions from dplyr work to restructure the data by changing the arrangement of rows and columns, or by modifying their contents. Here, we’ll look at two kinds of operation: mutate and bind.

Mutate

mutate is a useful function that creates new columns by calculating values from existing columns. For example, we can add a column of DBH values in inches this way:

treeData2<-mutate(treeData,DBHin=`DBH (cm)` / 2.54)

Here we’ve created a new variable called DBHin by multiplying all the values in DBH (cm) by 2.54. We can also create new values by combining values across multiple columns. For example, if we wanted to get crown depth, which is the difference between the total tree height and the crown height, we can subtract these using a mutate function:

treeData3<-mutate(treeData2,crownDepth_m=`TreeHt (m)`-`CrnHt (m)`)
ggplot(treeData3,aes(x=crownDepth_m)) + 
  geom_histogram()

The important thing to remember about these kinds of operations is that they are happening row-wise, so the values in the new column will correspond to the position of values in the columns used. For example, the first tree height value is 2 meters, while the first crown height value is 0.5. When the latter is subtracted from the former, we get a value of 1.5, and this becomes the first value in our new column.

Case_when

mutate is a very flexible tool for reworking data, but there are likely to be instances where you’ll want to apply different transformations depending on what the data are.

For example, let’s say we wanted to create a categorical variable that distinguished between two size classes: “big” trees being those more than 20 meters tall, and all others being “small” trees. Since we’re creating a new column from existing data, we know mutate is involved, but since we want to turn some values into “Big” and others into “Small”, we need something that expresses these rules.

The case_when function reads in values from a vector and, for cases where a given condition is met, it will return a given value. For example, let’s use it to check whether values in a vector are greater than or equal to 3:

x<-c(1,2,3,4,5)
case_when(
  x >= 3 ~ TRUE
)
[1]   NA   NA TRUE TRUE TRUE

Here, we see a formula as the first argument

x >= 3 ~ TRUE

Here, we’re asking R to look through the vector x we created and, for instances where the value is greater than or equal to 3 (>= 3), it will return a logical value of TRUE. Anything that is not covered by this condition receives an NA value.

If we didn’t want the NA values, we could supply an argument with another condition, or we can add a .default argument to cover anything not covered above:

case_when(
  x >= 3 ~ TRUE,
  .default = FALSE
)
[1] FALSE FALSE  TRUE  TRUE  TRUE

Knowing what this function gives back to us, we can combine this with mutate to create new columns based on these kinds of rules. For example, here we’ll create another vector of values and combine it with the first into a tibble:

y<-c(2,4,6,8,10)
data<-tibble(x,y)
data
# A tibble: 5 × 2
      x     y
  <dbl> <dbl>
1     1     2
2     2     4
3     3     6
4     4     8
5     5    10

We can then use mutate to create a logical column based on whether the values in a column meet a given criteria:

data2<-mutate(data,z = case_when(
  y >= 8 ~ TRUE,
  .default = FALSE
))
data2
# A tibble: 5 × 3
      x     y z    
  <dbl> <dbl> <lgl>
1     1     2 FALSE
2     2     4 FALSE
3     3     6 FALSE
4     4     8 TRUE 
5     5    10 TRUE

When we set the conditions in our case_when function, we’re not limited to drawing on a single column, but could use multiple columns in the formula. For example, let’s say we wanted to indicate rows where the value of y minus x is greater than or equal to 3:

data3<-mutate(data,z = case_when(
  y - x >= 3 ~ TRUE,
  .default = FALSE
))
data3
# A tibble: 5 × 3
      x     y z    
  <dbl> <dbl> <lgl>
1     1     2 FALSE
2     2     4 FALSE
3     3     6 TRUE 
4     4     8 TRUE 
5     5    10 TRUE

The output need not be logical, but could be numeric or character as well:

mutate(data3,w00t = case_when(
  z == TRUE ~ "w00t",
  .default = "Not w00t"
))
# A tibble: 5 × 4
      x     y z     w00t    
  <dbl> <dbl> <lgl> <chr>   
1     1     2 FALSE Not w00t
2     2     4 FALSE Not w00t
3     3     6 TRUE  w00t    
4     4     8 TRUE  w00t    
5     5    10 TRUE  w00t

OK, coming back to our original problem, we can create our two classes of trees (“Big” and “Small”) using a combination of mutate and case_when:

treeData4<-mutate(treeData3,sizeClass=case_when(
  `TreeHt (m)` > 20 ~ "Big",
  .default = "Small"
))
table(treeData4$sizeClass)

  Big Small 
 2026 12461

There’s nothing stopping us from using a single rule, either. For example, let’s say we wanted another class for “medium” trees between 10 and 20 meters tall. We can add another argument to case_when with a comma, and add our new condition there:

treeData4<-mutate(treeData3,sizeClass=case_when(
  `TreeHt (m)` > 20 ~ "Big",
  `TreeHt (m)` <=20 & `TreeHt (m)` > 10 ~ "Medium",
  .default = "Small"
))
table(treeData4$sizeClass)

   Big Medium  Small 
  2026   5793   6668

Bind

Sometimes you may want to combine two datasets into a single table. Let’s say we had two tables of different maple trees:

sugarMaples<-filter(treeData,CommonName=="Sugar maple")
redMaples<-filter(treeData,CommonName=="Red Maple")

We can put these two together using the bind_rows function, which we’ve seen before in our lab exercise using the Rarotonga weather:

mapleTrees<-bind_rows(sugarMaples,redMaples)

The bind_cols function works similarly, but instead binds new columns to an existing table. For example, let’s say we had our maple tree data in two pieces; information about streets and information species names:

mapleNames<-select(mapleTrees,ends_with("Name",ignore.case = TRUE))
mapleStreets<-select(mapleTrees,DbaseID,contains("Street",ignore.case = FALSE))

We can recombine these using the bind_cols function this way:

mapleData<-bind_cols(mapleStreets,mapleNames)

TipTry it yourself!

A distinguishing factor between the tidyverse bind_rows and its Base R equivalent (rbind) is that the Base R commands will only work with tables that have the same opposing dimension (same number of columns). The tidyverse versions will simply fill any unmatched columns or rows with NA values. Try it by combining the tree dataset with these datasets from the modeldata package:
